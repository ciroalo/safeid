# Domain model

This document describes the core domain objects used by SafeID.

The domain layer contains no external dependencies and represents the business logic of the application.

---

# Key Entities

## Image Asset

Represents decoded and normalized image data.

Fields: 

- `size_px`: image dimensions
- `mode`: pixel format (RGB)
- `pixel_bytes`: raw pixel data

Properties:
- No metadata
- Orientation already normalized
- Safe for deterministic rendering

--- 

## LayoutPlan

Describes how images should be placed on the page.

Fields:

- `page`: page specification
- `placed_images`: positions and sizes of images
- `photo_area`: bounding box of the images

This plan is generated by the layout planner and consumed by the renderer

---

## PlacedImage

Represents one image positioned on the PDF.

Fields:
- `source_index`: index of the image 
- `rect`: position and size
- `rotate_90`: indicates if the image is rotated and how many degrees

---

## RectPt

Rectangle defined in PDF points:

Fields:
- `x`: x position in the PDF
- `y`: y position in the PDF
- `width`: width of the rectangle
- `height`: height of the rectangle

Used for layout geometry

---

## PageSpec

Defines the physical page layout.

Fields:

- `page_size`: size of the page
- `margins`: margins of the PDF

MVP uses A4 with 19mm margins

---

## LayoutPolicy

Controls layout behavior.

Fields:

- `allow_rotation`: can the image be rotated
- `allow_scaling`: can the image be scaled
- `min_gap_mm`: min gap between images
- `stack_direction`: the direction of the stacked images

---

## WatermarkSpec

Describes the watermark appearance.

Fields:

- `text`: watermark text
- `font_family`: watermark text font
- `font_size_pt`: watermark font size in points
- `opacity`: watermark text opacity
- `angle_deg`: watermark text angle
- `line_spacing_pt`: spacing between watermark texts lines in points
- `word_spacing`: distance between words in a repeated line

---

# Domain Errors

Domain-specific erros provide clear failure semantics.

Examples:

- `InvalidImageCountError`
- `ImageTooSmallError`
- `UnsupportedFormatError`
- `ExportFailedError`

These error allow infrastructure failures to be mapped into user-facing messages.
